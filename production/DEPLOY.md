# Deploy City Café Rewards — Live URL

Deploy the full app (Customer + Staff POS + Admin + Neon DB) to a public URL using **Render** (recommended, free tier) and your existing **Neon** database.

Your live URL will look like:

**https://city-cafe-rewards.onrender.com**

---

## Architecture

```
Browser  →  Render (Node.js API + static UI)  →  Neon PostgreSQL (Singapore)
```

One service serves everything:

- `GET /` → Customer / Staff / Admin UI (`index.html`)
- `GET /health` → uptime check
- `POST /api/auth/login` → JWT login
- All other `/api/*` → protected API

## Step 2 — Deploy on Render (recommended)

1. Go to **[render.com](https://render.com)** → Sign up (free)
2. **New +** → **Blueprint**
3. Connect your GitHub repo `city-cafe-rewards`
4. Render detects `render.yaml` automatically
5. Click **Apply**

### Set environment variables

In Render → your service → **Environment**:

| Key | Value |
|-----|--------|
| `DATABASE_URL` | Your Neon connection string (from [neon.tech](https://neon.tech) dashboard) |
| `JWT_SECRET` | Auto-generated by Render, or paste: run `openssl rand -hex 32` |
| `NODE_ENV` | `production` (set by blueprint) |
| `JWT_EXPIRES_IN` | `7d` (set by blueprint) |

6. Click **Save Changes** → Render redeploys (~2–3 min)

### Your live URL

Render → service → **Settings** → copy URL, e.g.:

**https://city-cafe-rewards.onrender.com**

Open it in a browser. You should see the **login screen**.

> **Free tier note:** Render sleeps after ~15 min idle. First visit after sleep takes ~30 seconds to wake up.

## Neon — allow Render connections

Neon allows connections from anywhere by default (SSL required). No IP whitelist needed for Render.

Use the **pooled** connection string from Neon if available (better for serverless/free tiers).

Example:

```
postgresql://user:pass@ep-xxxx.ap-southeast-1.aws.neon.tech/neondb?sslmode=require
```

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `database: disconnected` | Check `DATABASE_URL` in Render env; must include `?sslmode=require` |
| Login fails | Run `npm run hash-passwords` against Neon from local machine |
| Blank page | Check Render logs; ensure repo includes `index.html` and `js/` at root |
| Slow first load | Render free tier cold start — upgrade to Starter ($7/mo) for always-on |
| 401 on all API calls | Log in first; JWT required for all `/api/*` except login/register |
