### Required environment variables (Render dashboard)

| Variable | Required | Notes |
|----------|----------|-------|
| `DATABASE_URL` | Yes | Neon connection string with `?sslmode=require` |
| `JWT_SECRET` | Yes | Auto-generated by Render or set manually |
| `NODE_ENV` | Yes | `production` |
| `JWT_EXPIRES_IN` | Optional | Default `7d` |
| `CORS_ORIGIN` | Optional | Only if using a custom domain |

**If `/health` returns `"database": "disconnected"`** → check `DATABASE_URL` in Render Environment.

## Known limitations

| Limitation | Notes |
|------------|-------|
| Render free tier cold start | ~30s delay after 15 min idle |
| Demo passwords | Not suitable for real café — change before launch |
| No payment gateway | Cash/card recorded manually at counter |
| No camera QR scan | Keyboard/scanner input only |
| Product images | Base64 in DB (prototype) |
| Single Render service | API + UI served together (by design) |
